# Stamply— MVP 기획서 수정안

### 📌 서비스 개요

> 팝업스토어 현장의 종이 브로슈어 + 스탬프 방식을 디지털화한 서비스.
관람객은 QR 스캔으로 미션을 수행하고, 운영자는 실시간으로 현황을 관리한다.
주요 플랫폼은 모바일웹(유저사이드) + PC웹(어드민사이드) + 랜딩페이지.
>

## ✅ MVP 핵심 기능

---

- **🌐 랜딩 페이지(2차 필수)**
    
    서비스 소개 페이지. 별도 로그인 없이 누구나 접근 가능하다.
    
    | 섹션 | 내용 |
    | --- | --- |
    | 히어로 섹션 | 서비스 이름, 슬로건, 시작하기 버튼 |
    | 서비스 소개 | Stamply가 무엇인지 간단한 설명 |
    | 주요 기능 소개 | QR 입장, 스탬프 미션, 완료 인증 등 핵심 기능 설명 |
    | 어드민 시작 버튼 | 행사 운영자용 어드민 로그인/ 회원가입 페이지 이동 |

---

### 👤 유저 페이지

---

**전체 플로우**

> QR 스캔 → 행사 입장 페이지 (포스터 + 시작하기) → 브로슈어 페이지 (스와이프) → 미션 페이지 → 
모든 미션 완료 → 설문조사 → 직원 확인 페이지
> 

---

**🚪 행사 입장**

관람객이 행사장 입구에 부착된 QR을 스캔하면 행사 입장 페이지로 이동한다. 별도 로그인 없이 임시 userId가 발급되며 쿠키 기반으로 저장된다. 브라우저를 닫아도 userId를 기반으로 진행 상태가 복원된다.

| 항목 | 내용 |
| --- | --- |
| 접속 방법 | 입장용 QR 스캔 |
| 화면 구성 | 행사 포스터 이미지 + 시작하기 버튼 + powered by Stamply |
| 유저 식별 | userId 발급 → 쿠키 저장 |
| 진행 복원 | 재접속 시 userId로 미션 완료 상태 복원 |

![qr후 행사 입장 페이지.png](attachment:96baf59a-494f-48aa-9f9d-534df463006a:95499c64-b50c-4f13-9261-6a82e0a43014.png)

---

**📖 브로슈어 페이지**

행사 입장 후 시작하기 버튼을 누르면 브로슈어 페이지로 이동한다. 행사 포스터 이미지를 슬라이드 형태로 제공하며, 스와이프하면 미션 페이지로 넘어간다. 미션 페이지 내에서도 우상단 맵 아이콘을 통해서는 브로셔 이미지와 행사정보를 볼 수 있다. 브로슈어 페이지는 있으면 보여주고 없으면 스킵한다. 

| 진입 경로 | 내용 |
| --- | --- |
| 시작하기 버튼 | 행사 입장 페이지 → 브로슈어 페이지로 이동 |
| 맵 아이콘 | 미션 페이지 우상단 아이콘 클릭 → 브로슈어 이미지 + 행사 상세보기 버튼 표시 |

| 항목 | 내용 |
| --- | --- |
| 브로슈어 이미지 | 어드민이 등록한 행사 포스터 이미지 표시 |
| 행사 상세보기 | 행사명, 운영 기간 및 시간, 행사 장소(지도보기), 주최측 정보(연락처) |
| 페이지 이동 | 좌우 스와이프로 미션 페이지 이동 / 터치 시 다음 페이지 이동 안내 표시 |

**🎯 미션 페이지**

행사 내 미션 목록을 보여주고 완료 여부를 표시한다. 스탬프(리스트)UI와 리스트UI 두 가지 뷰를 제공하며 우상단 아이콘으로 전환할 수 있다. 

| 항목 | 내용 |
| --- | --- |
| 상단 헤더 | 행사명 + 남은 스탬프 수 안내 (예: N개의 스탬프를 더 모으고 리워드를 받으세요) |
| UI 전환 | 우상단 아이콘으로 스탬프UI ↔ 리스트UI 전환 |
| 카드UI | 미션별 리스트 형태 — 스탬프 이미지 + 미션명 + 미션 설명 표시 |
| 스탬프UI | 미션별 스탬프를 그리드로 표시 — 완료 시 스탬프 이미지 활성화, 미완료 시 흐리게 표시 |
| 미션 상세 | 각 미션 리스트 클릭 시 해당 부스 상세 정보 표시 |
| QR 스캔 버튼 | 하단 고정 버튼 — 미션 미완료 시 `🔒 QR 체크하기` / 전체 완료 시 설문조사 모달 |
| 맵 아이콘 | 우상단 맵 아이콘 클릭 시 브로슈어 페이지로 이동 |

**📷 미션 완료 체크**

각 부스에 부착된 QR을 스캔하면 해당 미션이 완료 처리된다.

| 상황 | 처리 |
| --- | --- |
| 정상 스캔 | 미션 완료 처리 + 해당 미션 스탬프 활성화 |
| 이미 완료한 미션 | `이미 완료한 미션입니다` 표시 |
| 유효하지 않은 QR | `유효하지 않은 QR입니다` 표시 |
| 다른 행사 QR | `유효하지 않은 QR입니다` 표시 |

---

**🎁 완료 페이지**

모든 미션 완료 시 설문조사 UI가 모달로 표시된다. 제출 후 직원 확인 페이지로 이동하며, 해당 화면을 스태프에게 보여주면 리워드를 수령할 수 있다.

| 단계 | 내용 |
| --- | --- |
| 설문조사 | 성별 / 연령대 입력 후 제출하기 버튼 클릭(기본 설문은 4개(성별/연령대/만족도/유입경로)나 mvp는 2개로 고려) |
| 직원 확인 페이지 | Stamply 스탬프 이미지 + `리워드 수령처에서 이 화면을 스태프에게 보여주세요` 안내 + 직원 확인 버튼 |
| 중복 참여 시도 | 1. 완료된 채로 모든 페이지 다시 보여지게 3. mvp 2차때 qr로 인증 할거라 일단 임시 4. 시크릿 모드거나 다른 브라우저면 무한… |

**🎪 행사 관리**

행사의 기본 정보를 제공한다.

| 항목 | 내용 |
| --- | --- |
| 행사 CRUD | 생성(입장 qr도 생성됨) / 수정 / 삭제 - 운영기간에 따라 기능 제한 |
| 기본 정보 | 행사 이름, 날짜, 장소, 포스터 이미지, 브로셔 이미지 |
| 테마 적용 | 팔레트를 클릭해 행사 테마 컬러를 지정한다. 스탬프와 버튼등 행사 전반적인 ui에 적용 |
| 스탬프 모양 | 기본제공 혹은 주최측 이미지 추가 |
| 공지사항 | 운영시간, 주의사항 등록 |

**🎯 미션 관리**

| 항목 | 내용 |
| --- | --- |
| 미션 CRUD | 생성 / 수정 / 삭제 - 운영기간에 따라 기능 제한 |
| 부스 설명 | 각 미션별 상세 설명 등록 |
| QR 발급 | 각 미션별 QR 생성 및 출력 |

---

- **🎁 굿즈 수령 관리(현재 불필요)**
    
    관람객이 제시한 완료 화면을 확인하고 수령 처리한다.(현재 불필요)
    
    | 상황 | 처리 |
    | --- | --- |
    | 완료 화면 확인 | `수령 처리` 버튼 활성화 |

**📊 현황 조회**

| 항목 | 내용 |
| --- | --- |
| 전체 참여자 수 | 입장 QR 스캔 후 userId가 발급된 전체 수 |
| 미션 진행 중 | 미션을 1개 이상 완료했지만 전체 완료는 안 한 참여자 수 |
| 미션 전체 완료 | 모든 미션을 완료한 참여자 수 / % |
| 굿즈 수령 완료 | 설문 조사를 제출한 참여자 수 |
| 미션별 완료 현황 | 미션 이름 / 완료 수 / 완료율 / 이탈율 (%) |
| 시간대별 참여자 수 | 날짜별 / 시간대(1시간 단위) 기준 입장 참여자 수 / 피크 시간대 |
| 설문 현황 | 총 응답 수 / 성별·연령대 분포 (mvp 이후: 유입 경로 / 만족도 ) |

### ⚠️ 주요 고려사항

**MVP**

| 항목 | 내용 |
| --- | --- |
| 유저 식별 | 쿠키 기반 임시 userId (유니크키 + event_id 조합), 만료시간 = 행사 종료일 |
| 진행 복원 | 쿠키의 userId로 재접속 시 미션 완료 상태 복원 |

| 유저 라우트 보호 | 유저 라우트 보호미들웨어에서 쿠키의 userId 확인 → 입장 QR 미스캔 시 입장 안내 페이지로 리다이렉트, 미션 미완료 시 완료/설문 페이지 접근 차단 후 미션 페이지로 리다이렉트 |
