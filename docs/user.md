### 1-1. QR 스캔 → 행사 입장

관람객이 행사장 입구에 부착된 QR을 스캔하면 해당 행사 입장 페이지로 접속한다.  대표 컬러가 페이지 전체 팔레트에 적용된다. 별도 로그인 없이 쿠키 기반 임시 userId로 진행되며, 최초 접속 시 userId가 발급되어 쿠키에 저장된다.

> 💡 입장용 QR은 행사 전체에 하나만 존재한다. 부스별 QR(미션 완료용)과 다르다.
💡 입장용 QR URL 형태: `https://stamply.com/entry/{token}` → token으로 행사 식별
> 

**👤 유저 임시 아이디 관리**

- 쿠키로 관리한다
    - 쿠키 만료시간은 행사 기간 종료일로 설정한다
    - 유저 임시 아이디는 유니크키 + 행사 id 조합으로 구성한다
    - 행사 기간 내 5개 미션이 있을 때 첫날 2개, 둘째날 3개를 완료하는 시나리오도 가능하다

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 입장용 QR 스캔 | `https://stamply.com/entry/{token}` 접속 |
| 2 | token 검증 | Supabase `qr_codes`에서 token → event_id 조회 |
| 3 | 최초 접속 | 임시 userId 발급 → localStorage 저장 → Supabase `users` 생성 |
| 4 | 재접속 | localStorage에서 userId 조회 → Supabase에서 미션 진행 상태 복원 |
| 5 | 행사 입장 페이지로 이동 | - |

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 유효하지 않은 token | `유효하지 않은 QR입니다` |
| 존재하지 않는 행사 | `존재하지 않는 행사입니다` |
| 종료된 행사 | `종료된 행사입니다` |
| localStorage 없는 환경 | `지원 불가능한 브라우저입니다` |
| 네트워크 오류 | `네트워크 연결을 확인해주세요` |
| 입장 QR 미스캔 상태에서 미션 QR 스캔 | 입장 QR 스캔 안내 페이지로 리다이렉트 |

**보안 고려사항**

| 항목 | 내용 |
| --- | --- |
| userId 조작 방지 | Supabase RLS로 본인 데이터만 접근 가능하도록 제한 |

**사용 API**

```jsx
Supabase: qr_codes 조회        → token 검증 + event_id 조회
GET /{teamId}/goals/{goalId}  → 행사 정보 조회
Supabase: events 조회          → 행사 메타 정보
Supabase: users insert        → 최초 접속 시 userId 생성
Supabase: users 조회           → 재접속 시 진행 상태 복원
```

### 2-1. 브로슈어 페이지 조회

관람객이 행사 입장 페이지에서 시작하기를 누르면 보이는 페이지다. 어드민이 등록한 행사 포스터 이미지를 슬라이드 형태로 제공하며, 스와이프하면 미션 페이지로 이동한다. 지정된 컬러가 페이지 전체 팔레트에 적용된다.

**표시 정보**

| 항목 | 데이터 출처 |
| --- | --- |
| 브로슈어 이미지 | Supabase `events` (최대 개수 고려 필요) |
| 지정된 컬러 | Supabase `events` `color...` |

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 시작하기 버튼 클릭 | 브로슈어 페이지 로드 |
| 2 | 이미지 슬라이드 확인 | 등록된 브로슈어 이미지 표시 |
| 3 | 좌우 스와이프 또는 터치 | 미션 페이지로 이동 |

### 2-2. 행사 상세 조회

미션 페이지 우상단 맵 아이콘을 클릭하면 브로슈어 이미지와 행사 상세보기 버튼이 표시된다. 행사 상세보기 버튼을 누르면 어드민이 등록한 행사 상세 정보 및 공지사항을 조회할 수 있다.

**표시 정보**

| 항목 | 데이터 출처 |
| --- | --- |
| 브로슈어 이미지 | Supabase `events` |
| 포스터 이미지 | - |
| 행사명 | Goals API `title` |
| 운영 기간 및 시간 | Supabase `events` |
| 행사 장소 | Supabase `events` (지도보기 링크 포함) |
| 주최측 정보 | Supabase `events` (연락처 포함) |
| 운영시간 / 주의사항 | Supabase `events` (공지사항 없으면 섹션 미표시) |
| 대표 컬러 | Supabase `events` `primary_color` |

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 미션 페이지 우상단 맵 아이콘 클릭 | 브로슈어 이미지 + 행사 상세보기 버튼 표시 |
| 2 | 행사 상세보기 버튼 클릭 | 포스터 이미지 + 행사 상세 정보 + 공지사항 표시 |
| 3 | X 버튼 클릭 | 미션 페이지로 복귀 |

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 데이터 로드 실패 | `행사 정보를 불러오는데 실패했습니다. 다시 시도해주세요` |
| 브로슈어 이미지 없음 | 행사 상세보기 버튼만 표시 - 논의 필요 |
| 이미지 로드 실패 | 기본 이미지로 대체 -논의 필요 |
| 공지사항 없음 | 공지사항 섹션 미표시 |

**사용 API**

```jsx
GET /{teamId}/goals/{goalId}    → 행사명 조회
Supabase: events 조회           → 운영 기간 / 시간 / 장소 / 주최측 정보 / 브로슈어 이미지 / 운영시간 / 주의사항 / 대표 컬러 조회
```

### 3-1. 미션 목록 조회

관람객이 행사 내 미션 목록을 조회한다. 각 미션의 완료 여부와 전체 진행률을 스탬프UI 또는 리스트UI로 표시하며, 우상단 토글로 전환 가능하다. 기본UI는 스탬프UI다.

**표시 정보**

| 항목 | 데이터 출처 |
| --- | --- |
| 미션 목록 | Todos API (goalId 기준 필터) |
| 미션별 완료 여부 | Supabase `mission_completions` (userId 기준) |
| 전체 진행률 | 상단 문구 내 번호 바뀜 (애니메이션) |
| 스탬프 모양 / 대표 컬러 | Supabase `events` |

**UI 모드**

| 모드 | 설명 |
| --- | --- |
| 스탬프UI | 격자형으로 스탬프 표시, 완료 시 스탬프 활성화 |
| 리스트UI | 리스트형으로 미션 카드 표시, 완료 여부 표시 |

> 💡 UI 모드 선택은 localStorage에 저장해 재접속 시 유지된다.
> 

> 어드민이 미션을 비활성화(`done: true`)한 경우 → 비활성화된 미션은 목록에서 완전히 숨김
> 

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 미션 페이지 접속 | 미션 목록 + 완료 여부 + 진행률 로드 + 하단 QR 체크하기 버튼 표시 |
| 2 | 우상단 토글 클릭 | 스탬프UI ↔ 리스트UI 전환 |
| 3 | 우상단 맵 아이콘 클릭 | 행사 상세 조회 → 2-2 참고 |
| 4 | 미션 항목 클릭 | 미션 상세 모달 표시 → 3-2 참고 |
| 5 | 하단 QR 체크하기 버튼 클릭 | QR 스캔 화면 활성화 |
| 6 | QR 스캔 완료 | 해당 미션 스탬프 활성화 + 진행률 업데이트 |
| 7 | 모든 미션 완료 시 | 설문조사 모달 바로 표시 |

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 데이터 로드 실패 | 텅UI |
| 미션 없음 | 텅UI |
| 유효하지 않은 QR | `유효하지 않은 QR입니다` |
| 이미 완료한 미션 | `이미 완료한 미션입니다` |
| 다른 행사 QR 스캔 | `유효하지 않은 QR입니다` |

**사용 API**

```jsx
GET /{teamId}/todos                  → 미션 목록 조회 (goalId 필터)
Supabase: mission_completions 조회    → userId 기준 미션 완료 여부
Supabase: qr_codes 조회               → token 검증
Supabase: mission_completions insert → 미션 완료 기록 (userId + todoId)
Supabase: events 조회                 → 스탬프 모양 / 대표 컬러 조회
```

### 3-2. 미션 상세 조회

관람객이 미션 항목을 클릭하면 모달(뒤에 오버레이)로 해당 부스의 상세 설명을 확인할 수 있다.

**표시 정보**

| 항목 | 데이터 출처 |
| --- | --- |
| 미션 이름 | Todos API `title` |
| 부스 상세 설명 | Notes API `content` |
| 완료 여부 | Supabase `mission_completions` (userId 기준) |

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 미션 항목 클릭 | 미션 상세 모달 표시 (미션 이름 / 부스 상세 설명) |
| 2 | 오버레이 배경 클릭 또는 뒤로가기 | 모달 닫힘 → 미션 목록으로 복귀 |

> ⚠️ **크로스 브라우저 이슈**: 오버레이 배경 클릭 시 모달 닫힘 처리, 오버레이 뒤 화면 스크롤 차단 (iOS Safari / Android Chrome 대응 필요)
> 

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 데이터 로드 실패 | `미션 정보를 불러오는데 실패했습니다. 다시 시도해주세요` |
| 부스 설명 없음 | 설명 섹션 미표시 |

**사용 API**

```jsx
GET /{teamId}/todos/{todoId}    → 미션 상세 조회
GET /{teamId}/notes             → 부스 상세 설명 조회 (todoId 기준)
Supabase: mission_completions 조회 → 완료 여부 확인
```

### 4-1. QR 스캔 → 미션 완료 처리

관람객이 부스에 부착된 QR을 스캔하면 해당 미션이 완료 처리된다. QR 스캔은 모바일 브라우저 카메라를 통해 이루어지며, 스캔된 토큰의 유효성을 검증한 후 미션 완료를 기록한다.

> 💡 QR에는 고유 토큰이 인코딩되어 있다. 스캔 시 해당 토큰을 수파베이스 `qr_codes` 테이블에서 조회해 유효성을 검증한다.
> 

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | QR 스캔 버튼 클릭 | 카메라 권한 요청 → 카메라 활성화 |
| 2 | 부스 QR 스캔 | 토큰 추출 → 유효성 검증 |
| 3 | 유효한 토큰 확인 | 미션 완료 처리 → 진행률 업데이트(상단)→ 완료 애니메이션 표시 |
| 4 | 전체 미션 완료 시 | `모든 미션을 완료했습니다! 굿즈를 수령하세요` 안내 + 설문조사 모달 |

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 카메라 권한 거부 | `카메라 권한이 필요합니다. 설정에서 권한을 허용해주세요` |
| 이미 완료한 미션 QR | `이미 완료한 미션입니다` |
| 유효하지 않은 QR | `유효하지 않은 QR입니다` |
| 다른 행사 QR 스캔 | `유효하지 않은 QR입니다` |
| 비활성화된 QR (재발급된 경우) | `유효하지 않은 QR입니다` |
| 네트워크 오류 | `네트워크 연결을 확인해주세요` |
| iOS Safari 카메라 미지원 | `브라우저에서 카메라를 지원하지 않습니다. Safari를 사용해주세요` |

**중복 방지 로직**

> session_id + todo_id 조합이 `mission_completions` 테이블에 이미 존재하면 완료 처리하지 않는다.
> 

**사용 API**

```jsx
Supabase: qr_codes 조회              → token 유효성 확인 + todo_id 추출
Supabase: mission_completions 조회   → userId + todo_id 중복 확인
Supabase: mission_completions insert → 미션 완료 기록 (userId + todo_id)
```

### 5-1. 완료 후 설문조사

모든 미션을 완료한 관람객에게 설문조사 모달이 표시된다. 설문 완료 후 직원 확인 페이지로 이동하며, 해당 화면을 스태프에게 보여주면 리워드를 수령할 수 있다.

> 💡 설문조사 모달은 모든 미션을 완료한 경우에만 표시된다. 미션이 남아있는 상태에서 직접 URL로 접근하면 미션 페이지로 리다이렉트된다.
> 

**표시 정보**

| 항목 | 내용 |
| --- | --- |
| 성별 | 남 / 여 선택 |
| 연령대 | 연령대 선택 |

> 2차 mvp에서 만족도와 유입경로 직접입력 고려
> 

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 전체 미션 완료 | 설문조사 모달 자동 표시 |
| 2 | 설문조사 작성 | 성별 / 연령대 입력 |
| 3 | 모든 항목 입력 완료 | 제출 버튼 활성화 |
| 4 | 제출하기 버튼 클릭 | 직원 확인 페이지로 이동 |

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 미션 미완료 상태에서 직접 URL 접근 | 미션 페이지로 리다이렉트 |
| 이미 설문 완료 후 재접속 | 직원 확인 페이지 그대로 복원 |
| 설문 항목 미입력 | 제출 버튼 비활성화 |
| 네트워크 오류 | `네트워크 연결을 확인해주세요` |

**사용 API**

```jsx
Supabase: mission_completions 조회    → 전체 미션 완료 여부 확인
Supabase: users update                → 설문 응답 저장
```

### 5-2. 직원 확인 페이지

설문조사 제출 후 관람객에게 리워드 수령 안내 화면을 표시한다. 화면을 스태프에게 보여주면 리워드를 수령할 수 있다.

**표시 정보**

| 항목 | 내용 |
| --- | --- |
| 브랜드 로고 | 스탬플리 로고 혹은 스탬프 로고? |
| 완료 메시지 | `고생하셨습니다! 리워드 수령하러 고고🎉` |
| 수령 안내 | `리워드 수령처에서 이 화면을 스태프에게 보여주세요` |
| 직원 확인 버튼 | 스태프가 직접 클릭하여 수령 처리 |

> 🔜 **2차 필수**: 완료 화면 토큰 기반 QR 생성 → 어드민 스캔으로 수령 처리 / 또는 직원 확인 버튼 고도화

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 설문 완료 전 직접 URL 접근 | 미션 페이지로 리다이렉트 |
| 이미 수령한 계정 재접속 | 완료 페이지 보일지.. |
| 네트워크 오류 | `네트워크 연결을 확인해주세요` |

**사용 API**

```jsx
Supabase: users 조회     → 설문 완료 여부 확인
Supabase: events 조회    → 리워드 수령 장소 안내
```
