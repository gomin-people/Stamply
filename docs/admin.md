### 1-1. 로그인

어드민 페이지 진입 시 카카오 로그인으로 인증한다. 로그인 성공 후 대시보드 또는 생성 페이지로 이동한다.

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 카카오 로그인 클릭 | 카카오 인증 → 토큰 발급 |
| 2 | 행사가 1개인 경우 | 해당 행사 대시보드로 바로 이동 |
| 3 | 행사가 2개 이상인 경우 | 최근 행사 대시보드로 바로 이동 |
| 4 | 행사가 없는 경우 | 행사 생성 페이지로 이동 |

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 카카오 인증 실패 | `카카오 로그인에 실패했습니다. 다시 시도해주세요` |

**권한 체크 플로우**

> 로그인 → API 토큰 발급 → Supabase `user_roles` 조회 → `role: admin` 확인 → 행사 수 확인 → 행사 1개면 대시보드 / 2개 이상 최근 행사 대시보드 / 없으면 행사 생성 페이지 이동 - 수정 필요

### 2-1. 행사 생성

어드민이 새 행사를 생성한다. 생성 완료 시 입장용 QR과 URL이 자동 생성된다.

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 행사 이름 입력 | - |
| 2 | 시작 날짜 / 종료 날짜 / 장소 입력/ 주최측 정보(연락처) | - |
| 3 | 포스터 이미지 업로드 | - |
| 4 | 브로셔 이미지 업로드(선택사항) | - |
| 5 | 스탬프 모양 선택 (기본 제공 1개 or 커스텀 SVG 업로드) | - |
| 6 | 대표 컬러 선택 (선택, 미선택 시 기본 컬러 자동 적용) |  |
| 7 | 운영시간 / 공지 사항 입력 | - |
| 8 | 생성 버튼 클릭 | 입장용 QR + URL 자동 생성 → 해당 행사 대시보드로 이동 |

**저장 구조**

| 데이터 | 저장 위치 |
| --- | --- |
| 행사 이름 | Goals API `title` |
| 시작 날짜 / 종료 날짜 / 장소 / 포스터 이미지 URL / 스탬프 모양 /  대표 컬러 / 운영시간 / 공지 사항 / 주최 이름/주최 연락처/ 입장 URL / 입장 QR | Supabase `events` |

💡 대표 컬러는 브로슈어 페이지 + 미션 페이지 + 스탬프에 전체 팔레트가 적용된다.

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 행사 이름 미입력 | `행사 이름을 입력해주세요` |
| 시작 날짜 미입력 | `행사 시작 날짜를 입력해주세요` |
| 종료 날짜 미입력 | `행사 종료 날짜를 입력해주세요` |
| 종료 날짜가 시작 날짜보다 앞인 경우 | `종료 날짜는 시작 날짜 이후여야 합니다` |
| 장소 미입력 | `행사 장소를 입력해주세요` |
| 대표 컬러 미선택 | 기본 컬러 자동 적용 |
| 이미지 업로드 실패 | `이미지 업로드에 실패했습니다. 다시 시도해주세요` |
| API 실패 | `행사 생성에 실패했습니다. 다시 시도해주세요` |
| Supabase 저장 실패 | 롤백 후 `행사 생성에 실패했습니다. 다시 시도해주세요` |
| QR 생성 실패 | `QR 생성에 실패했습니다. 다시 시도해주세요` |

**사용 API**

```jsx
POST /{teamId}/goals          → 행사 이름 저장
POST /{teamId}/images         → 포스터 이미지 업로드
Supabase: events insert       → 메타 정보 + 입장 URL / QR 저장
```

### 2-2. 행사 수정

어드민이 등록된 행사의 정보를 수정한다. 수정 버튼 클릭 시 기존 데이터를 불러와 폼에 채워주고, 수정 완료 후 저장한다. 운영 상태에 따라 수정 가능 범위가 다르다.

**운영 상태별 수정 가능 범위**

| 상태 | 수정 가능 항목 |
| --- | --- |
| 운영 전 | 전체 수정 가능 |
| 운영 중 | 행사 이름 / 장소 / 운영시간 / 주의사항만 수정 가능 (스탬프 모양 / 대표 컬러 수정 불가) |
| 운영 후 | 수정 불가 (조회만 가능) |

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 수정 버튼 클릭 | 기존 행사 데이터 폼에 자동 입력 |
| 2 | 수정할 항목 변경 | 운영 상태에 따라 수정 가능 항목 제한 |
| 3 | 저장 버튼 클릭 | Goals API → Supabase events 업데이트 → 해당 행사 대시보드로 이동 |

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 행사 이름 비워두고 저장 | `행사 이름을 입력해주세요` |
| 종료 날짜가 시작 날짜보다 앞인 경우 | `종료 날짜는 시작 날짜 이후여야 합니다` |
| 데이터 불러오기 실패 | `데이터를 불러오는데 실패했습니다. 다시 시도해주세요` |
| 이미지 업로드 실패 | `이미지 업로드에 실패했습니다. 다시 시도해주세요` |
| 수정 실패 | `수정에 실패했습니다. 다시 시도해주세요` |
| 운영 중 / 운영 후 스탬프 모양 수정 시도 | 스탬프 모양 / 대표 컬러 항목 비활성화 |

**사용 API**

```jsx
GET /{teamId}/goals/{goalId}          → 기존 행사 이름 조회
PATCH /{teamId}/goals/{goalId}        → 행사 이름 수정
GET Supabase: events                  → 기존 메타 정보 조회
POST /{teamId}/images                 → 포스터 이미지 재업로드 (변경 시)
Supabase: events update               → 메타 정보 수정
```

### 3-1. 미션 생성

어드민이 행사에 포함될 미션을 등록한다. 각 미션은 부스 하나에 대응되며, 미션 이름과 부스 상세 설명을 함께 등록한다. 미션 생성 완료 시 해당 미션의 QR이 자동 생성된다. 부스 상세 설명은 Notes API에 저장되며 `todoId`로 연결된다.

> 📌 미션 순서 안내
미션은 생성 순서로 고정 표시됩니다. 순서 변경이 필요한 경우 삭제 후 재생성해주세요.
> 

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 미션 이름 입력 | - |
| 2 | 부스 상세 설명 입력 (리치 텍스트) | - |
| 3 | 생성 버튼 클릭 | - |

**저장 구조**

| 데이터 | 저장 위치 | 필드 |
| --- | --- | --- |
| 미션 이름 | Todos API | `title` |
| 행사 연결 | Todos API | `goalId` (필수값으로 전달) |
| 부스 상세 설명 | Notes API | `content` (ProseMirror JSON), `todoId`로 연결 |
| 미션 QR | Supabase `qr_codes` | `todo_id` 연결 |

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 미션 이름 미입력 | `미션 이름을 입력해주세요` |
| 미션 9개 초과 생성 시도 | `미션은 최대 9개까지 등록할 수 있습니다` |
| 미션 0개인 상태에서 행사 오픈 | `미션을 1개 이상 등록해야 합니다` |
| Todos API 실패 | `미션 생성에 실패했습니다. 다시 시도해주세요` |
| Notes API 실패 | Todos 롤백 후 `미션 생성에 실패했습니다. 다시 시도해주세요` |
| QR 생성 실패 | `QR 생성에 실패했습니다. 다시 시도해주세요` |

**사용 API**

```jsx
POST /{teamId}/todos          → 미션 생성 (goalId 필수 포함)
POST /{teamId}/notes          → 부스 상세 설명 저장 (todoId 연결)
Supabase: qr_codes insert     → 미션 QR 자동 생성
```

### 3-2. 미션 수정

어드민이 등록된 미션의 내용을 수정한다. 수정 버튼 클릭 시 기존 데이터를 불러와 폼에 채워주고, 수정 완료 후 저장한다. 운영 상태에 따라 수정 가능 범위가 다르다.

**운영 상태별 수정 가능 범위**

| 상태 | 수정 가능 항목 |
| --- | --- |
| 운영 기간 전 | 미션 이름 / 부스 상세 설명 전체 수정 가능 |
| 운영 중 | 부스 상세 설명만 수정 가능 
미션 비활성화 가능(UI)/ db는 done으로 표시 |
| 운영 후 | 수정 불가 (조회만 가능) |

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 수정할 미션 선택 | 기존 미션 데이터 폼에 자동 입력 |
| 2 | 미션 이름 / 부스 상세 설명 수정 | 운영 상태에 따라 수정 가능 항목 제한 |
| 3 | 저장 버튼 클릭 | Todos API → Notes API 순서로 업데이트 → 미션 목록으로 이동 |

**미션 비활성화 플로우 (운영 중)**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 미션 비활성화 버튼 클릭 | 확인 모달 표시(ui는 바뀔수 있음) |
| 2 | 모달 확인 | `done: true` 업데이트 → 해당 미션 비활성화 상태로 표시 |

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 미션 이름 비워두고 저장 | `미션 이름을 입력해주세요` |
| 데이터 불러오기 실패 | `데이터를 불러오는데 실패했습니다. 다시 시도해주세요` |
| 수정 실패 | `수정에 실패했습니다. 다시 시도해주세요` |
| 운영 중 미션 이름 수정 시도 | `운영 중에는 미션 이름을 수정할 수 없습니다` |
| 운영 후 수정 시도 | `종료된 행사의 미션은 수정할 수 없습니다` |

**사용 API**

```jsx
GET /{teamId}/todos/{todoId}      → 기존 미션 데이터 조회
GET /{teamId}/notes/{noteId}      → 기존 부스 상세 설명 조회 (todoId로 연결)
PATCH /{teamId}/todos/{todoId}    → 미션 이름 수정 (운영 전만)
PATCH /{teamId}/todos/{todoId}    → done: true (미션 비활성화, 운영 중만 가능, 어드민 비활성화 전용)
PATCH /{teamId}/notes/{noteId}    → 부스 상세 설명 수정
```

### 3-3. 미션 삭제

어드민이 등록된 미션을 삭제한다. 삭제 시 해당 미션에 연결된 부스 설명(Notes)과 QR 코드도 함께 삭제된다. 실수 방지를 위해 삭제 확인 모달을 표시한다. 삭제는 운영 기간 전에만 가능하다.

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 삭제 버튼 클릭 | `이 미션을 삭제하시겠습니까? 삭제된 미션은 복구할 수 없습니다` 모달 표시 |
| 2 | 확인 클릭 | 미션 + 부스 설명 + QR 삭제 → 미션 목록으로 이동 |
| 3 | 취소 클릭 | 모달 닫기 |

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| 삭제 실패 | `삭제에 실패했습니다. 다시 시도해주세요` |
| 운영 중 / 운영 후 삭제 시도 | 삭제 버튼 비활성화 |

**사용 API**

```jsx
DELETE /{teamId}/todos/{todoId}    → 미션 삭제
DELETE /{teamId}/notes/{noteId}    → 부스 설명 삭제
Supabase: qr_codes delete          → 해당 미션 QR 삭제
```

### 3-4. 미션별 QR 생성 및 출력

미션 생성 시 자동 생성된 QR을 출력하거나 재생성한다. QR에는 고유 토큰이 인코딩된 URL이 포함되며, 관람객이 스캔하면 해당 미션 완료 처리 페이지로 이동한다.

> 💡 QR은 미션 생성 시 자동 생성된다. 어드민은 생성된 QR을 출력해 부스에 부착하고, 관람객이 현장에서 스캔해 미션을 완료하는 구조다.
> 

**정상 플로우**

| 단계 | 행동 | 결과 |
| --- | --- | --- |
| 1 | 미션 목록에서 QR 확인 | 미션 생성 시 자동 생성된 QR 이미지 표시 |
| 2 | QR 출력 / 다운로드 버튼 클릭 | 프린트 다이얼로그 표시 or PNG 파일 다운로드 |
| 3 | QR 재생성 버튼 클릭 | 확인 모달 표시 |
| 4 | 모달 확인 | 기존 QR 삭제 → 새 token 생성 → 새 QR 이미지 표시 |
| 5 | 모달 취소 | 모달 닫기 |

**QR 스캔 흐름** 

| 단계 | 내용 |
| --- | --- |
| QR 인코딩 값 | `https://stamply.com/mission/{token}` |
| 스캔 시 | 해당 URL로 이동 → token 검증 → user_id 확인 → 미션 완료 처리 |
| 유효한 토큰 | `mission_completions`에 user_id + todo_id insert → 미션 완료 처리 |
| 무효한 토큰 | `유효하지 않은 QR입니다` 표시 |
| 이미 완료한 미션 | `이미 완료한 미션입니다` 표시 |

**QR 생성 흐름**

| 단계 | 내용 |
| --- | --- |
| 1 | token 기반 URL 조합 → `https://stamply.com/{type}/{token}` |
| 2 | qrcode 라이브러리로 QR 이미지 실시간 생성 |
| 3 | canvas or img 태그로 화면 표시 |
| 4 | 출력 버튼 클릭 시 프린트 다이얼로그 표시 |
| 5 | 다운로드 버튼 클릭 시 PNG 파일로 저장 |

> 💡 QR 이미지는 Supabase에 저장하지 않고 프론트에서 token 기반으로 실시간 생성
💡 다운로드 / 출력 모두 프론트에서 처리 가능 (qrcode 라이브러리 PNG 다운로드 지원)
> 

---

**저장 구조**

```jsx
Supabase qr_codes
├── id          PK
├── todo_id     미션 연결 (FK, nullable - 입장 QR일 경우 null)
├── event_id    행사 연결 (FK, nullable - 미션 QR일 경우 null)
├── token       고유 토큰 (QR URL 조합에 사용)
├── type        'mission' | 'entry' | 'reward' (추후 확장 시)
└── created_at
```

**예외 케이스**

| 상황 | 처리 |
| --- | --- |
| QR 생성 실패 | `QR 생성에 실패했습니다. 다시 시도해주세요` |
| QR 재생성 시 기존 토큰 무효화 실패 | `QR 재생성에 실패했습니다. 다시 시도해주세요` |

**사용 API**

```jsx
Supabase: qr_codes delete    → 기존 QR 삭제
Supabase: qr_codes insert    → 새 토큰 생성
```

- 바로 되는지 아니면 어느 페이지로 보낼지 바로 페이지 될수 있는지 (qr찍었을경우)

### 4-1. 참여자 현황

어드민이 행사의 실시간 참여 현황을 조회한다. PC에서는 대시보드 메인에 전체 현황을 표시한다.

**표시 정보**

| 항목 | 내용 |
| --- | --- |
| 전체 참여자 수 | 입장 QR 스캔 후 userId가 발급된 전체 수 |
| 미션 진행 중 | 미션을 1개 이상 완료했지만 전체 완료는 안 한 참여자 수 |
| 미션 전체 완료 | 모든 미션을 완료한 참여자 수 / % |
| 굿즈 수령 완료 | 설문 조사를 제출한 참여자 수 |

---

### 4-2. 미션별 완료율

어드민이 각 미션의 완료율을 조회한다. 어떤 미션에서 관람객이 많이 이탈하는지 파악할 수 있다.

**표시 정보**

| 항목 | 내용 |
| --- | --- |
| 미션 이름 | 각 미션 제목 |
| 완료 수 | 해당 미션을 완료한 참여자 수 |
| 완료율 | 전체 참여자 대비 완료 비율 (%) |
| 이탈율 | 전체 참여자 대비 미완료 비율 (%) |

**예시**

| 미션 | 완료 수 | 완료율 | 이탈율 |
| --- | --- | --- | --- |
| 1번 부스 체험 | 120명 | 80% | 20% |
| 2번 부스 체험 | 90명 | 60% | 40% |
| 3번 부스 체험 | 45명 | 30% | 70% |

**사용 API**

```jsx
GET /{teamId}/todos                   → 미션 목록 조회 (어드민 세팅용)
Supabase: mission_completions 조회     → todo_id 기준 미션별 완료 수 집계
Supabase: users 조회                   → 전체 참여자 수 (완료율 계산용)
```

### 4-3. 시간대별 참여자 현황

입장 시간 기준으로 날짜별 / 시간별 참여자 수를 조회한다.

**표시 정보**

| 항목 | 내용 |
| --- | --- |
| 날짜별 참여자 수 | 날짜 기준 입장 참여자 수 집계 |
| 시간대별 참여자 수 | 시간대(1시간 단위) 기준 입장 참여자 수 집계 |
| 피크 시간대 | 참여자가 가장 많은 시간대 표시 |

**사용 API**

---

```jsx
Supabase: users 조회        → created_at 기준 날짜별 / 시간별 집계
```
