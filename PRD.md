## 1. 개요 (Overview)

### 1.1 제품 요약

**Stamply**는 팝업 스토어와 전시 행사 방문객을 위한 **디지털 스탬프 적립 및 설문조사 플랫폼**이다. 종이 스탬프카드를 모바일 웹으로 대체하고, 행사 운영사는 방문객 데이터를 수집·분석할 수 있으며, 방문객은 미션 완료를 통해 상품을 수령할 수 있다.

### 1.2 핵심 가치 제안

**행사 운영사:**

- 방문객 행동 데이터 실시간 수집 및 분석
- 간편한 미션·설문조사 관리
- 상세한 통계 및 인사이트 대시보드

**방문객:**

- 디지털 스탐프로 분실 걱정 없음
- 미션 완료 후 간단한 설문으로 상품 수령
- 행사 정보 및 브로슈어 한눈에 확인

## 2. 문제 정의 (Problem Statement)

| 대상        | 현재 문제                                       | Stamply 해결책                        |
| ----------- | ----------------------------------------------- | ------------------------------------- |
| 행사 운영사 | 종이 스탬프카드 관리 어려움, 방문객 데이터 부재 | 디지털 관리 + 실시간 통계 대시보드    |
| 행사 운영사 | 설문조사 수집 및 분석 번거로움                  | 모웹 내 설문조사 자동화 + 보고서 생성 |
| 방문객      | 스탬프카드 분실, 위·변조 위험                   | 모바일 웹 기반 디지털 스탐프          |
| 방문객      | 미션 완료 후 상품 수령 절차 복잡                | 설문 완료 후 현장에서 즉시 수령       |

## 3. 대상 사용자 (Target Users)

### 3.1 행사 운영사 (B2B)

**페르소나:**

- 팝업 스토어, 브랜드 전시회, 문화 행사 등을 운영하는 담당자
- 30~50대, 기본적인 디지털 literacy 보유

**핵심 니즈:**

- 방문객 데이터 수집 및 분석
- 간편한 미션/설문조사 운영
- 실시간 통계 확인

**사용 기기:** 웹 브라우저 (데스크톱/태블릿)

### 3.2 방문객 (B2C)

**페르소나:**

- 20~50대 도시 거주자
- 팝업/전시 자주 방문하는 사람
- 스마트폰 기본 사용 능력 보유

**핵심 니즈:**

- 미션 완료 후 상품 수령
- 간편한 모웹 조작 (별도 설치 불필요)
- 분실 걱정 없는 디지털 스템프

**사용 기기:** 스마트폰 브라우저 (Chrome, Safari)

## 5. 사용자 플로우 (User Flows)

### 5.1 행사 운영사 플로우

```
로그인
→ 행사 등록 (행사명, 기간, 장소, 브로슈어)
→ 미션 생성 (1~10개, QR 자동 생성)
→ 설문조사 설정 (필수항목 + 커스텀 1개)
→ 행사 시작
→ 실시간 대시보드 모니터링
→ 행사 종료
```

### 5.2 방문객 플로우

#### 최초 방문객

```
행사장 입구에서 입장 QR 스캔 (기기 기본 카메라 사용)
→ 브라우저 자동 이동
→ 임시 회원번호 발급 (쿠키 저장)
→ 행사 정보 화면
→ "참여하기" 클릭
→ 브로슈어 (있으면 슬라이드 보기, 없으면 건너뜀)
→ 미션 리스트 페이지
```

#### 미션 완료 플로우

```
미션 1 클릭 → 상세 내용 확인 → 닫기 → 미션 목록 "QR 버튼" 클릭
→ 부스 담당자에게 QR 제시받아 스캔
→ 미션 1 완료 표시
→ 미션 2, 3, ... 반복
→ 모든 미션 완료
→ 완료 화면 노출 + "설문조사하기" 버튼 활성화
```

#### 설문조사 및 상품 수령 플로우

```
"설문조사하기" 클릭
→ 설문 페이지 (필수 4개 + 커스텀 1개)
→ 별점 5개로 만족도 선택
→ 저장
→ 완료 화면
→ 하단 "관리자 확인" 버튼
→ 현장 관리자가 클릭
→ DB 업데이트 + 알럿 표시
→ 상품 수령 완료
```

#### 재방문객

```
같은 브라우저/기기로 다시 방문
→ 입장 QR 스캔
→ 쿠키 확인 (이전 임시 회원번호 있음)
→ 중복 상품 수령 방지
→ 새로운 행사면 새 임시 회원번호 발급
```

## 6. 관리자 대시보드 상세 사양

### 6.1 주요 지표 (Summary Cards)

```
┌─────────────────────────────────────────────────────┐
│ 전체 참여자    │ 미션 진행    │ 미션 완료    │ 상품 수령  │
│   1,234명      │   850명      │   650명      │   520명    │
└─────────────────────────────────────────────────────┘
```

- **전체 참여자**: 입장 QR 스캔한 사람
- **미션 진행**: 최소 1개 이상 미션 완료한 사람
- **미션 완료**: 모든 미션을 완료한 사람
- **상품 수령**: 현장에서 "관리자 확인" 처리 완료한 사람

### 6.2 미션 현황

**미션별 완료율 테이블**

```
미션명          │ 완료자  │ 완료율
─────────────────────────────
포토존          │ 1,050명 │ 85.2%
댓글이벤트      │   890명 │ 72.2%
상품 구매        │   670명 │ 54.3%
```

**부스별 완료율 그래프** (막대 그래프)

- X축: 미션명
- Y축: 완료율 (%)
